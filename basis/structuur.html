Basisstructuur van een webpagina

Met behulp van HTML, HyperText Markup Language, beschrijf je de algemene structuur en de globale opmaak van webpagina's.

Het uiterlijk van een webpagina is mede afhankelijk van de interpretatie die de browser aan de HTML codes geeft. Het is bijvoorbeeld niet ongebruikelijk dat dezelfde HTML pagina er in Chrome of Firefox er anders uitziet dan in Internet Explorer!

Een webpagina is opgebouwd uit zogenaamde HTML tags die de browser vertellen hoe de inhoud van het document op het scherm gezet moet worden en hoe bijvoorbeeld Hyperlinks naar andere documenten gemaakt kunnen worden.

Dit is een voorbeeld van een simpele HTML pagina:

<!DOCTYPE html> <html> <head> <title>Tussen deze tags plaats je de titel van de webpagina</title> </head> <body> Tussen deze tags komt de inhoud van de webpagina </body> </html>

Websites ontwikkelen

Om zelf webpagina's te maken heb je een teksteditor nodig om in te programmeren. Je kan hiervoor de standaard Windows kladblok gebruiken. Maar het is een stuk fijner om een uitgebreidere editor te gebruiken zoals TextMate of Notepad++. Die voegen kleurtjes toe aan de HTML waarmee je makkelijker gemaakte fouten kan vinden.

HTML programmeren is zo simpel als de HTML opslaan in een .html bestand en dit bestand weer in een browser te openen. Let op dat wijzigingen die je in het bestand maakt niet automatisch in de browser zullen verschijnen. Je moet eerst het bestand opslaan en daarna op 'verversen' klikken in de browser.

Valideren

Bij Programmeren 1 heb je misschien gemerkt dat Java klaagt over elke puntkomma en leesteken die je niet goed zet. HTML is op dat gebied precies het tegenovergestelde. Browsers proberen HTML zo goed mogelijk weer te geven, ook al staat deze vol met fouten.

Dit is natuurlijk geen reden om slordige HTML te schrijven. Om te testen of jouw HTML zich 100% aan de standaard houd kan je gebruik maken van de HTML validator. Met dit online tooltje kun je controleren of je geen fouten hebt gemaakt in je HTML.
