Formulieren

Vooraf

In de voorgaande weken hebben we de belangrijkste mogelijkheden besproken die HTML biedt om informatie te ordenen, op te maken en aan te bieden in de vorm van webpagina's. Gewapend met deze kennis kunnen we complete websites maken waarin de bezoekers, al naar gelang hun voorkeur, die pagina's bekijken die voor hen relevant zijn.

Vaak willen we echter ook informatie van de bezoeker hebben om daar vervolgens op te kunnen reageren. Tot nu toe was er slechts sprake van eenrichtingsverkeer op onze website; de datastroom van de webserver naar de client.

Om ook met de bezoekers van onze website te kunnen "praten" en dus een interactieve website te maken, hebben we in HTML formulieren nodig. Formulieren kunnen voor veel verschillende doeleinden gebruikt worden. De belangrijkste categoriën zijn: feedback en webapplicaties.

In het eerste geval kunnen wij de bezoekers om een mening vragen, bijvoorbeeld over onze website. De opmerkingen die we op deze manier binnenkrijgen kunnen we bijvooorbeeld gebruiken om onze website te verbeteren. In het tweede geval vragen wij onze bezoekers gegevens in te vullen die door een webapplicatie (die meestal op de server draait) verwerkt worden. Denk hierbij bijvoorbeeld aan het plaatsen van een bestelling bij een webwinkel.

In deze cursus kijken wij niet naar de mogelijkheden om deze gegevens te verwerken (dit gebeurt in de vervolgcursus) maar gaan wij na welke mogelijkheden formulieren allemaal bieden om gegevens aan te leveren.


Een pagina met een formulier maken

De HTML-tags waarmee je een formulier maakt zijn: <form> en </form>.
Dit zijn de basistags voor het maken van formulieren en tussen deze twee codes plaats je alle overige codes voor de opmaak van het formulier. Net zoals vele andere HTML elementen kent ook de formulier tag diverse attributen.
Twee daarvan, de attributen method en action, ben je min of meer verplicht om te gebruiken. Met behulp van deze attributen kun je namelijk specificeren op welke wijze de ingevulde gegevens naar de webserver gestuurd worden én op welke wijze deze gegevens verwerkt worden. Alléén in het geval dat je de ingevulde gegevens niet op de server verwerkt maar bijvoorbeeld met JavaScript op de client kun je deze attributen achterwege laten.
Met behulp van het attribuut method geef je aan hoe de gegevens naar de server gestuurd moeten worden. Vrijwel altijd krijgt dit attribuut de waarde "post". Hiermee geef je aan dat de browser de ingevulde gegevens naar de server verstuurt.
Met behulp van het attribuut action geef je aan hoe de verstuurde gegevens verwerkt moeten worden. Dat wil zeggen dat je hiermee aangeeft welk programma en/of welk script op welke server de gegevens verwerkt.

Samengevat ziet de minimale openingstag voor een formulier er dus als volgt uit: 
  <form
    method="post"
    action="http://www.mijnwebsite.nl/cgi-bin/mijnprogramma/">
Achter action hebben we dus aangegeven naar welke server de gegevens verstuurd worden. Namelijk de server waarop "mijnwebsite" draait. Daarnaast heb je aangegeven dat de gegevens verwerkt moeten worden door het programma "mijnprogramma" dat zich bevindt in de map cgi-bin. (Het is overigens een goede gewoonte op internet om dit soort programma's in deze map te plaatsen.)

Er is nog een andere mogelijkheid voor het action attribuut. Je kunt nl. ook aangeven dat de gegevens naar een email-adres gestuurd moeten worden. De inhoud van het action attribuut wordt dan als volgt: 
  <form
    method="post"
    action="mailto:janjansen@provider.nl?subject=formulierX"
    enctype="text/plain">
Je ziet dat er i.p.v. het http-protocol gebruik gemaakt wordt van het mailto-protocol. Dit wordt gevolgd door het emailadres waar de gegevens naar toe verstuurd moeten worden en (evt) een omschrijving die in de onderwerp-regel van de email boodschap geplaatst moet worden. Tenslotte wordt via het enctype-attribuut aangegeven dat de gegevens als gewone ASCII-tekst verstuurd mag worden.
In de body van de gegenereerde email boodschap worden de respectievelijke velden van het formulier met de waardes die de gebruiker heeft ingevuld weergegeven.


Formulieren inrichten

De tag <input>
De eerste vraag die we ons moeten stellen als we een formulier gaan opmaken heeft betrekking op het soort invoervelden dat we in het formulier willen opnemen. We kunnen onze bezoekers bijvoorbeeld een keuze uit een menu laten maken of zelf een tekst laten intikken als antwoord op een vraag. De verschillende mogelijkheden die er zijn worden hieronder kort besproken.

De belangrijkste tag waarmee we het soort invoerveld van een formulier definiëren is de <input> tag. Deze tag staat natuurlijk binnen de form tags en kent twee verplichte attributen: name en type.

Met het name attribuut van de input tag geef je het invoerveld een naam. Dit is noodzakelijk omdat je anders bij het verwerken van de gegevens niet weet van welk veld de gegevens afkomstig zijn.

Met het type attribuut van de input tag specificeer je om wat een soort invoerveld het gaat. De meest gebruikte waarden van het type attribuut zijn:

Text: Hiermee creëer je een tekstinvoerveld van één regel.
Bij het gebruik van dit type invoerveld zul je vaak ook de attributen size en maxlength van de input tag gebruiken om respectievelijk de lengte van het invoerveld zoals dat op het scherm wordt weergegeven, vast te stellen en het maximaal aantal karakters dat ingevoerd mag worden te bepalen. De code voor een invoerveld van dit type, waarin de gebruiker bijvoorbeeld zijn postcode kan invoeren, ziet er dan als volgt uit: 
<input name="postcode" type="text" size="6" maxlength="6">
Tenslotte kun je bij dit type invoerveld ook nog het attribuut value gebruiken om een standaard tekst (default waarde)in het invoerveld te plaatsen.
Password: Dit type is hetzelfde als het type text met dien verstande dat de ingevoerde tekst als sterretjes op het scherm verschijnen.
Checkbox: Een invoerveld van dit type zet een selectievakje op scherm dat de gebruiker kan selecteren. Als je meerdere velden van dit type in een document plaatst kan de gebruiker deze in principe allemaal selecteren. Als je wilt dat een gebruiker een keuze maakt uit een aantal alternatieven gebruik je een radiobutton of een keuzelijst. Zie hieronder.
Door het attribuut checked op te nemen bij een van de input tags wordt het betreffende vakje standaard geselecteerd.
Radio: Een invoerveld van dit type wordt gebruikt om de gebruiker een keuze uit een aantal alternatieven te laten maken. Denk bijvoorbeeld aan een meerkeuze vraag.
Om de browser te laten weten welke invoervelden van het type radio bij elkaar horen, geef je deze velden allemaal dezelfde naam! Om toch te kunnen weten welk veld gekozen is, gebruik je bij het type radio ook het attribuut value. Met behulp van dit attribuut kun je in dit geval toch ieder veld een unieke naam geven. Tenslotte kun je ook hier bij een van de input tags het attribuut checked opnemen om een standaardwaarde te selecteren.
Button: Met behulp van dit type kun je een button op het scherm plaatsen waarop de gebruiker kan klikken en dat vervolgens bijvoorbeeld een script opstart.
Submit: Met behulp van dit type invoerveld creër je een button in je document waarop de gebruiker kan klikken om de ingevulde gegevens te versturen. Zonder deze knop in het formulier kunnen de gegevens dus niet verstuurd en verwerkt worden! Deze button neem je dus altijd op in je formulier.
Met behulp van het attribuut value kun je de button een betekenisvolle naam in je formulier geven. Wil je bijvoor een submit button in je formulier opnemen met de tekst verstuur gegevens dan neem je daarvoor de volgende code in je formulier op: 
      <input type="submit" value="Verstuur gegevens">
Reset: Met behulp van dit type invoerveld creër je een button in je document waarop de gebruiker kan klikken om de ingevulde gegevens van alle invoervelden te wissen. Het is een goede gewoonte om een dergelijke knop ook altijd in je formulier op te nemen. Evenals bij de submit button kun je voor deze knop ook zelf een naam bedenken die je opgeeft bij het attribuut value.



De tag <select>
Hierboven hebben we gezien hoe we met behulp van de input tag diverse soorten invoervelden kunnen creëren. Een andere veel gebruikte tag om een invoerveld in een formulier op te nemen is de select tag in combinatie met de option tag.
Met behulp van deze tags kunnen we een menu in ons formulier opnemen waaruit de gebruiker een keuze kan maken. De algemene vorm om een keuzemenu in je formulier op te nemen ziet er dan als volgt uit: 
  <select>
    <option>keuze 1</option>
    <option>keuze 2</option>
    <option>keuze 3</option>
    <option>keuze 4</option>
  </select>
Het zal geen verbazing wekken dat ook deze tags een aantal attributen kennen waarmee we de opmaak nader kunnen specificeren. Hieronder worden de belangrijkste attributen kort besproken.

Attributen van de select tag
Name: Met behulp van dit attribuut geef je de keuzelijst een unieke naam. Deze naam wordt tesamen met de gemaakte keuze als een naam/waarde paar naar de server gestuurd. Hoewel het gebruik van dit attribuut formeel niet verplicht is, gebruik je het altijd om de gemaakte keuze(s) te kunnen identificeren en te verwerken.
Size: Met de waarde die je bij dit attribuut opgeeft, specificeer je het aantal items dat tegelijkertijd zichtbaar is in het keuzemenu.
Multiple: Met het gebruik van dit element kun je aangeven dat er meer dan een item geselecteerd kan worden.

Attributen van de option tag
Value: Met behulp van dit attribuut kun je opgeven welke waarde naar de server gestuurd moet worden als het betreffende item geselecteerd is. Standaard wordt de waarde die in de keuzelijst staat naar de server gestuurd.
Selected: Hiermee kun je een item uit de keuzelijst als standaardwaarde selecteren die in het keuzemenu zichtbaar moet zijn.

Ter verduidelijking volgt hieronder een voorbeeld. Bekijk de volgende code: 
  <form method="post" action="mijnwebserver/MijnProgramma">
  <select name="voorbeeldmenu" size="3" multiple>
    <option value="k1">keuze 1</option>
    <option value="k2">keuze 2</option>
    <option value="k3" selected>keuze 3</option>
    <option value="k4">keuze 4</option>
    <option value="k5">keuze 5</option>
    <option value="k6">keuze 6</option>
  </select>
  </form>

Toelichting:
  1. Met behulp van de form tags geven we aan dat we een formulier willen maken.
  2. Met behulp van de attributen post en action van de form tag specificeren we hoe de gegevens verstuurd moeten worden en met welk programma de gegevens op de server verwerkt moeten worden.
  3. Met de select tag geven we aan dat we een keuzemenu in het formulier willen opnemen.
  4. Met het attribuut name van de select tag geven we het keuzemenu een naam ter identificatie van de ingestuurde gegevens.
  5. Met het attribuut size van de select tag specificeren we hoeveel items in het keuzemenu zichtbaar zijn. In dit voorbeeld zijn dat er dus drie.
  6. De option tag gebruiken we om een menu item te maken. Voor ieder menu item gebruiken we een aparte option tag.
  7. Met het attribuut value van de option tag specificeren we de waarde die naar de server gestuurd wordt als het desbetreffende item geselecteerd is. In het voorbeeld is dat dus "k1", "k2" etc.. Indien we dit attribuut niet gebruiken wordt de lijstwaarde van het gekozen item naar de server gestuurd. Dat zou in dit geval dus zijn "keuze 1", "keuze 2" etc..
  8. Met het attribuut selected van de option tag geven we tenslotte aan dat keuze 3 als standaardwaarde geselecteerd moet zijn.

De tag <textarea>
Een derde tag naast de input en de select tag waarmee we een invoerveld voor een formulier kunnen definiëren is de tag "textarea".

Met behulp van de tag textarea kun je een tekstvenster in je formulier opnemen. Deze tag is vooral handig als je je bezoekers relatief veel tekst wilt laten invoeren. Denk bijvoorbeeld aan het antwoord op een open vraag. Met de attributen cols en rows van deze tag specificeer je de afmetingen van het tekstveld. Met het attribuut name geef je de textarea een naam.
Tenslotte kun je nog een tekst tussen de begin en sluit tag typen. Deze tekst wordt dan standaard in het tekstveld geplaatst en je bezoekers kunnen die vervolgens natuurlijk wijzigen.

Samengevat ziet de HTML code voor het maken van een "tekstinvoervenster" er als volgt uit: 
  <textarea name="tekstinvoervenster" cols="60" rows="15">
    Deze tekst verschijnt als standaardtekst in het venster.
  </textarea>
