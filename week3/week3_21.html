XHTML, deel 1

Inleiding

HTML verkeert momenteel in versie 4.01. Het W3C is nog niet klaar met de mogelijke opvolger: HTML 5. Maar wel is HTML inmiddels omgezet naar 'een toepassing' van XML (eXtensible Markup Language), waarbij het de nieuwe naam XHTML krijgt wat staat voor eXtensible Hypertext Markup Language.

Het W3C realiseerde zich namelijk, dat - net als in het verleden gebeurd is - men met HTML nooit in staat zou zijn om aan de vragen vanuit de web community te voldoen naar (steeds weer) meer en nieuwe manieren om documenten te distribueren, verwerken en tonen.
HTML kent maar een beperkte set aan mogelijkheden om documenten te maken.
Probleemgebieden zijn b.v.:

  • niet-traditionele content
    • chemische formules
    • wiskundige expressies
    • muziek-notatie
  • alternatieve display media
    • handheld computers
    • intelligente telefoons


In plaats van zelf steeds nieuwe uitbreidingen op HTML te formuleren en te standaardiseren, heeft het W3C ervoor gekozen om HTML te herformuleren en het daarbij te laten voldoen aan de XML-standaard, omdat het XML ziet als de basis voor de toekomst van het Web.
We willen echter niet de massa aan bestaande HTML-documenten "weggooien" - of ze nu netjes in elkaar gezet zijn of niet -, net zo min als de bestaande kennis, tools en technologieën die HTML ondersteunen. Aan de andere kant willen we de mogelijkheden van XML ook gaan benutten.
XHTML is een beetje de brug tussen beide werelden. Het gebruikt de features van XML om een markup language te definiëren die vrijwel identiek is aan standaard HTML 4.01 en die ons toch laat beginnen met XML.

XML-intro

Een XML document bevat character data, die uit rechttoe rechtaan inhoud bestaat en uit markup (opmaak) in de vorm van tags en XML declaraties. 

  <?xml version="1.0" encoding="UTF-8"?>
  <blah>harrumph</blah>
De regels hierboven vormen in principe een well-formed XML document.
Well-formed XML documenten volgen bepaalde regels, zoals de verplichting dat iedere (begin-)tag een sluit-tag moet hebben en dat je tags netjes moet nesten.
Bovendien zie je dat XML documenten beginnen met een regel waarin aangegeven wordt dat er XML in het document staat, welke versie XML het is (hier 1.0) en dat de 8-bit Unicode character set gebruikt wordt (één van de meest gebruikte).

Een ander voorbeeld: zie vragen_01.xml.

Om een valid XML document te zijn moet de inhoud naast well-formed zijn ook nog aan een soort XML declaratie voldoen. Die XML declaratie legt vast hoe de tags en de inhoud er (ongeveer) uit moeten zien.
Deze declaratie(s) kunnen in het XML-document zelf opgenomen zijn, maar - zoals zo vaak - is het i.v.m. herbruikbaarheid handiger om ze in een apart bestand op te nemen.
Er zijn 2 smaken van dergelijke declaratie bestanden: een Schema of een DTD. We kijken hier alleen naar een DTD: een Document Type Definition.

Verwijzing naar een DTD bestand regel je door (bijna) vooraan in het XML-bestand een regel als de volgende op te nemen: 

  <!DOCTYPE blah SYSTEM "blah.dtd">
Zie het volgende leden.xml bestand voor een voorbeeld.
Hier hoort leden.dtd als Document Type Definition bij.

XHTML Document Type Definitions

XHTML kent 3 variantenen, die eigenlijk overeenkomen met 3 varianten die ook voor HTML 4.01 bestaan, maar waar we het nooit over gehad hebben. Bij iedere variant hoort een eigen DTD.
Om een XHTML document te maken moet je 1 van deze 3 DTDs kiezen en dan het document vullen volgens de regels van de gekozen DTD.
De volgende 3 DTDs zijn mogelijk:

  1. strict
  2. transitional
  3. frames

De 1e XHTML DTD correspondeert dus met de strict HTML DTD.
Deze strikte definitie sluit alle deprecated elementen (tags and attributes) in HTML 4.01 uit en dwingt de maker om alleen die features te gebruiken die volledig ondersteund worden in HTML.
Veel van de HTML elementen en attributen die met presentatie en voorkomen te maken hebben, zoals de <font> tag en het align-attribuut, mogen dan niet gebruikt worden. In plaats daarvan moeten de tegenhanger uit het CSS model gebruikt worden.

Je geeft als volgt in een XHTML document aan dat de strict DTD gebruikt moet worden: 

  <!DOCTYPE html 
     PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN"
     "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">
Met dit statement declareer je dat het root element van je document html is, zoals die gedefinieerd wordt in de DTD wiens publieke identifier "-//W3C//DTD XHTML 1.0 Strict//EN" is. De browser weet wellicht hoe de DTD te vinden die overeenkomt met deze publieke identifier, en als dat niet het geval is, dan kan het de URL gebruken die daarna opgegeven wordt als een alternatieve locatie voor de DTD.

Veel HTML auteurs vinden de strict XHTML DTD te beperkend, omdat veel van de deprecated elementen en attributen nog frequent gebruikt worde. Sterker nog, de populaire browsers ondersteunen soms nog niet alle standaard elementen, terwijl alle deprecated elementen wel volledig ondersteund worden.
Het enige echte voordeel van de strict XHTML DTD is dat die documenten gegarandeerd volledig ondersteund zullen worden in toekomstige versies van XHTML.

Veel auteurs zullen derhalve kiezen voor de transitional XHTML DTD. Deze staat het dichtste bij de huidige HTML standaard en bevat toch al die mooie maar deprecated features die het leven van een HTML auteur soms zo makkelijk maken. Met de transitional XHTML DTD kun je langzaamaan de overstap naar de XML familie maken (vandaar de naam transitional) en ondertussen toch de huidige browsers ten volle gebruiken.

Je geeft als volgt in een XHTML document aan dat de transitional DTD gebruikt moet worden: 

  <!DOCTYPE html 
     PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
     "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">


De 3e DTD is voor frames. Deze is identiek aan de transitional DTD, behalve dat de document body vervangen wordt door frame-elementen.
Je zou mogelijk verwachten, dat er voor "completeness"s sake", ook een strict en een transitional frame DTD zouden zijn, maar het W3C heeft besloten dat als je frames gebruikt, dan mag je net zo goed alle deprecated elementen gebruiken.

Je geeft als volgt in een XHTML document aan dat de transitional DTD gebruikt moet worden: 

  <!DOCTYPE html 
     PUBLIC "-//W3C//DTD XHTML 1.0 Frameset//EN"
     "http://www.w3.org/TR/xhtml1/DTD/xhtml1-frameset.dtd">


Een minimaal XHTML document ziet er dus als volgt uit:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE html 
    PUBLIC "-//W3C//DTD XHTML 1.0 Transitional//EN"
    "http://www.w3.org/TR/xhtml1/DTD/xhtml1-transitional.dtd">
<html>
  <head>
    <title>Every document must have a title</title>
  </head>
  <body>
    ...your content goes here...
  </body>
</html>
