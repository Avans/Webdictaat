JavaScript, een inleiding

Vooraf

Zoals de naam al doet vermoeden is JavaScript een scripttaal. Dit betekent o.a. dat de JavaScript code niet gecompileerd wordt naar een binaire code maar wordt "geïnterpreteerd" door de browser van de client en vervolgens wordt uitgevoerd. Dit in tegenstelling tot bijvoorbeeld java applets die platform onafhankelijk gecompileerd zijn en zonder verdere "interpretatie" van de browser uitgevoerd worden.

Zowel Java als JavaScript zijn objectgeoriënteerde talen. Dit betekent dat je in JavaScript de code schrijft voor het manipuleren van objecten. De standaardobjecten die JavaScript kent, hebben o.a. betrekking op de componenten waaruit een webpagina is opgebouwd. Door met behulp van JavaScript opdrachten voor deze objecten te schrijven kun je de functionaliteit en het uiterlijk van webpagina's beïnvloeden. Naast het manipuleren van bestaande objecten kun je in JavaScript ook zelf objecten maken.

Zoals we weten uit Java zijn objecten opgebouwd uit methoden en eigenschappen. De methoden die bij een object horen specificeren wat je met dat object kunt doen. De eigenschappen hebben betrekking op de verschijningsvorm, de karakteristieken van het object. Een voorbeeld van een JavaScript object is het object window. Dit object kent een aantal eigenschappen zoals status en name en een aantal methoden waaronder alert.

Met de eigenschap status kun je aangeven wat voor een tekst er in de statusbalk van het browservenster moet verschijnen. Hiermee beïnvloed je dus de verschijningsvorm (wel of geen tekst in de statusbalk) van het actuele venster. (Om het ingewikkeld te maken: een eigenschap van een bepaald object kan zelf ook een object zijn met zijn eigen methoden en eigenschappen. Hierop komen we later terug.)

Met de methode alert van het object window kun je een waarschuwingsvenster openen en in het actuele browservenster weergeven. In tegenstelling tot een eigenschap kun je met een methode dus iets doen, iets uitvoeren of iets maken en niet alleen maar invloed uitoefenen op de verschijningsvorm of de karakteristieken die een object heeft.

 

JavaScript, de taal in het (heel) kort

Syntax-eigenschappen:

  1. Case-sensitive
  2. Statements scheiden door puntkomma, maar is niet verplicht aan einde van regel
  3. Statements groeperen met accolades
  4. Commentaar als Java: // of /* ... */

Gegevenstypes:

  1. String
  2. Getal
  3. Boolean
  4. Null: volledig zonder waarde
  5. NaN: Not a Number
  6. Object
  7. Functie: functiedefinitie

Variabelen:

  1. Declareren en (evt) initialiseren via var, niet verplicht maar wel aan te bevelen
  2. Geen declaratie van type (i.t.t. b.v. Java)
  3. Variabelen mogen van type veranderen (!!): lekker flexibel, maar ook foutgevoelig

Expressies & Operatoren: zie js_evaluator.html.

Controle-structuren:

  1. if (voorwaarde) {statement(s)}
  2. if (voorwaarde) {statement(s)} else {statement(s)}
  3. switch (expression) {
    case label: statement; break;
    case label : statement; break;
    default : statement;}
  4. for ([beginexpressie];[voorwaarde];[bijwerkexpressie]) {statement(s)}
  5. do {statement(s)} while (condition)
  6. label/break/continue
  7. for (variable in object) {statement(s)}
  8. with (object) {statement(s)}

Functies:

  1. Definitie: function functieNaam ([parm1]...[parmN]) {statement(s)}
  2. Parameters zonder type-opgave
  3. Parameters worden doorgegeven by value, behalve objecten (by reference)
  4. Variabelen gedefinieerd binnen functie zijn lokaal, erbuiten globaal

Arrays:

  1. Declaratie: var a = new Array(5) --> arrays zijn objecten
  2. Array literal: coffees = ["French Roast", "Columbian", "Kona"]
  3. Index start bij 0

 

JavaScript in een webpagina opnemen

Er is een aantal manieren waarop je JavaScript code in je pagina kan opnemen.

  1. Als statements en functies tussen de <script> en </script> tags.

  2. Door te verwijzen naar een bestand met de JavaScript code
    <script src="http://www.mijnwebsite.nl/javascript.js">

  3. Als event handler binnen html tags
    <form ... onSubmit="mijnFunctie(); ">
    <a href="http://mijnURL" onClick="mijnFunctie();">

  4. Door een JavaScript expressie op te nemen als waarde voor een html attribuut.
    <form ... onReset="javascript:expressie;">

De eerste methode gebruik je als je JavaScript code alleen van toepassing is voor een enkele HTML pagina van je website. De JavaScript code neem je op tussen de script tags <script> en </script>. De script tags kunnen op iedere plaats en zo vaak als je wilt in je html-document opgenomen worden. Waar je de script tags in je document opneemt is afhankelijk van wat de code moet doen. Het is wel een goede gewoonte om de JavaScript functies die je schrijft in de head van je html-document op te nemen.

De tweede methode gebruik je als je JavaScript code van toepassing is op meerdere html-pagina's van je website. De JavaScript functies die je gebruikt kun je in een apart bestand op de server plaatsen. In de HTML pagina's waarin je die functies nodig hebt, hoef je dan slechts een verwijzing naar dat bestand op te nemen.

De derde methode gebruik je om een JavaScriptfunctie aan te roepen nadat een event heeft plaatsgevonden. Zie voor een toelichting van events en eventhandlers de paragraaf hieronder.

Vanwege de leesbaarheid van je HTML en JavaScript code wordt de vierde manier om JavaScript in je document op te nemen afgeraden.

 

Events, eventhandlers

JavaScript functies worden nooit zomaar uitgevoerd. Er moet altijd een gebeurtenis (event) zijn die de opdracht veroorzaakt/aanroept. Voor het detecteren van deze events kennen we zogenaamde event handlers. Dit zijn de specifieke gebeurtenissen die kunnen optreden waarmee een stuk JavaScript aangeroepen cq in werking gesteld kan worden. Gebeurtenissen kunnen bijvoorbeeld het klikken van de muis op een knop zijn of het laden van een webpagina. Met behulp van de event-handler onload kun je dus bijvoorbeeld onmiddelijk een stuk JavaScript uitvoeren nog voordat de pagina geladen is. De eventhandler is een attribuut van het HTML-element waarop je met JavaScript wilt reageren.

Zoals we in de inleiding al gezien hebben wordt JavaScript nooit zomaar uitgevoerd. Aan de uitvoering van JavaScript ligt altijd een gebeurtenis (event) ten grondslag. Om te detecteren of een event zich voordoet en om vervolgens een JavaScript op te starten kent HTML zogenaamde event handlers. Event handlers zijn eigenschappen van HTML elementen en je neemt ze derhalve op als attributen van de relevante HTML tag.
